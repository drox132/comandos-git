Estos son comandos comunes de Git que se usan en varias situaciones:

iniciar un área de trabajo (ver también: tutorial de ayuda de git) clon Clonar un repositorio en un nuevo directorio init Crea un repositorio Git vacío o reinicializa uno existente

trabajar en el cambio actual (ver también: ayuda de git todos los días) add Agregar contenido de archivo al índice mv Mover o cambiar el nombre de un archivo, un directorio o un enlace simbólico restaurar Restaurar archivos de árbol de trabajo rm Eliminar archivos del árbol de trabajo y del índice

examinar el historial y el estado (ver también: revisiones de ayuda de git) bisect Utilice la búsqueda binaria para encontrar la confirmación que introdujo un error diff Muestra los cambios entre confirmaciones, confirmación y árbol de trabajo, etc. grep Imprime líneas que coinciden con un patrón log Mostrar registros de confirmación showMostrar varios tipos de objetos estado Mostrar el estado del árbol de trabajo

crecer, marcar y modificar su historia común sucursal Enumerar, crear o eliminar sucursales commit Registrar cambios en el repositorio fusionar Unir dos o más historias de desarrollo juntas rebase Reapply confirma sobre otro consejo base restablecer Restablecer HEAD actual al estado especificado cambiar cambiar ramas etiqueta Crear, enumerar, eliminar o verificar un objeto de etiqueta firmado con GPG

colaborar (ver también: flujos de trabajo de ayuda de git) buscar Descargar objetos y referencias de otro repositorio extraiga Fetch e integre con otro repositorio o una sucursal local empujar Actualizar referencias remotas junto con objetos asociados

'git help -a' y 'git help -g' enumeran los subcomandos disponibles y algunos guías conceptuales. Consulte 'git help ' o 'git help ' para leer sobre un subcomando o concepto específico. Consulte 'git help git' para obtener una descripción general del sistema.

Comando Argumentos Que hace
git config --global user.name "Francisco Burgos" sirve para setear el valor del nombre del usuario a todos los proyectos en general
git config --global user.email "2013fjbh@gmail.com" sirve para setear el valor del email del usuario en todos los prectos
git config --local es lo mismo que el global pero para la foma local
git init para iniciar un repositorio local
git add nombreArchivo para agregar un archivo especifico
git add . se agrega todos los archivos al stayin area
git status estado del repositorio local
git commit -m "comentario" coloca un comentario al repositorio descriptio de lo que se hizo
git log trae el historial del commit completo
